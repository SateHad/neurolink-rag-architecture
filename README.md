# NeuroLink Omni-Channel RAG Engine

A production-ready Retrieval-Augmented Generation (RAG) routing pipeline engineered for automated client communication. Built on n8n, this system ingests real-time webhooks from Meta Graph APIs (Instagram & Messenger), executes semantic context retrieval via Supabase, and features stateful spam protection with dynamic human-in-the-loop failover.

![NeuroLink RAG Architecture](architecture.png)

## 🧠 System Architecture

1. **Ingestion & Verification:** Direct webhook endpoint configured to parse stateless incoming payloads from Meta Graph APIs. Includes automated security routines to authenticate connection signatures.
2. **Stateful Session Isolation & Anti-Spam:** Employs a defensive execution branch to protect system resources and LLM token budgets. When active support is flagged, the session is temporarily isolated, suppressing automated model inferences for a two-hour window to maintain clean single-agent communication boundaries.
3. **Vector Retrieval (RAG):** Utilizes Supabase (PostgreSQL + pgvector) to execute semantic similarity matches on ingested documentation pools, dynamically preserving context across multi-turn conversation frames.
4. **Dynamic Human-in-the-Loop Failover:** Fallback modules scan model outputs for systemic exceptions or explicit user escalation requests. Triggers instant communication relays (via Gmail API) to notify human agents with complete state data.

## 🛠 Tech Stack
* **Orchestration:** n8n (Self-Hosted/VPS or Cloud)
* **Database & Vector Store:** Supabase (PostgreSQL with `pgvector`)
* **AI & Embeddings:** OpenAI Models, Cohere Reranking
* **Integrations:** Meta Graph API, Gmail API

## 🚀 Deployment Guide

This repository contains the workflow schema and architectural blueprint. To deploy this pipeline, you must configure your own infrastructure and supply your own API credentials.

### 1. Database Setup (Supabase)
Before importing the workflow, you must initialize your Supabase PostgreSQL database with the required schema and `pgvector` extension.

Run the following SQL commands in your Supabase SQL Editor:

```sql
-- Enable vector extension
CREATE EXTENSION IF NOT EXISTS vector WITH SCHEMA public;

-- 1. Chat History Table
CREATE TABLE public.n8n_chat_histories (
    id integer NOT NULL GENERATED ALWAYS AS IDENTITY,
    session_id character varying(255) NOT NULL,
    message jsonb NOT NULL,
    PRIMARY KEY (id)
);

-- 2. Anti-Spam / Ban Tracking Table
CREATE TABLE public."n8nNeuroLinkBans" (
    id bigint NOT NULL GENERATED BY DEFAULT AS IDENTITY,
    "Timestamp" time without time zone,
    "Unban_Time" time without time zone,
    "Ban_Date" date,
    "Unban_Date" date,
    PRIMARY KEY (id)
);

-- 3. Vector Embeddings Table
CREATE TABLE public.neurodata (
    id bigint NOT NULL GENERATED ALWAYS AS IDENTITY,
    content text,
    metadata jsonb,
    embedding public.vector(1536),
    PRIMARY KEY (id)
);

-- 4. Vector Search Function (RPC)
CREATE FUNCTION public.match_documents(query_embedding public.vector, match_count integer DEFAULT 10, filter jsonb DEFAULT '{}'::jsonb) 
RETURNS TABLE(id bigint, content text, metadata jsonb, similarity double precision)
LANGUAGE sql STABLE
AS $$
    SELECT
        id,
        content,
        metadata,
        1 - (embedding <=> query_embedding) AS similarity
    FROM public.neurodata
    WHERE metadata @> filter
    ORDER BY embedding <=> query_embedding
    LIMIT match_count;
$$;
```

### 2. n8n Environment Configuration
1. Spin up an n8n instance (e.g., via Hostinger KVM VPS using Docker, or n8n Cloud).
2. Open the n8n UI, navigate to **Workflows**, and click **Import from File**.
3. Select the `workflow.json` file from this repository.
4. You will need to authenticate the nodes by providing your own credentials for:
   * **OpenAI API** (for LLM and Embeddings)
   * **Supabase API** (URL and Service Role Key)
   * **Meta Graph API** (System User Access Tokens)
   * **Gmail API** (For human-handoff email alerts)

### 3. Meta Webhook Configuration
1. Go to the Meta Developer Dashboard and configure a new App.
2. Set up Webhooks for Messenger and Instagram.
3. Point the Webhook URL to your active n8n production webhook endpoint.
4. Ensure the Verify Token matches the logic inside the initial n8n verification node.
